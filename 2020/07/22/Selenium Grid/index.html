1. 整个流程

要使用Selenium Grid进行分布式的操作，步骤大致是下面几个。
  1. 在本地CMD创建hub
  2. 在hub创建node,要对node配置端口号
  3. 在python脚本中配置要执行哪一个Node

    webdriver 是用来执行并操作浏览器的一个API，程序脚本通过webdriver来直接对浏览器进行操作，相当于起到一个链接浏览器和程序的桥梁。

3. Selenium Grid启动

什么是Selenium Grid呢？

如果你想要在多台机器上同时启动多个浏览器进行Web应用程序的测试，那么你就需要使用 Selenium Grid。其有一个hub和若干个node节点组成。hub用来管理各个node的注册信息，接受远程客户端的请求调用，然后将相应的请求转发给node来执行。
简单来说就是通过Selenium Grid可以实现在一台电脑上控制多台node节点来远程执行selenium自动化测试脚本。自动化测试脚本是放在hub主机上的，那么通过grid就可以控制下属的node客户端来执行hub主机上的自动化测试脚本。

3.1 启动hub

java -jar selenium-server-standalone-3.149.0.jar -port 4455 -role hub

-port 4455是开启4455号端口-role hub是将Selenium server作为hub或者Grid server

3.3 使用json文件来自定义节点

在使用json自定义节点前，必须先开启hub

cmd输入命令java -jar selenium-server-standalone-3.149.0.jar -role node -nodeConfig nodeconfig.json
nodeConfig.json文件如下

{
  "capabilities":
  [
    {
      "browserName": "firefox",
      "maxInstances": 3,
      "seleniumProtocol": "WebDriver"
    },
    {
      "browserName": "chrome",
      "maxInstances": 5,
      "seleniumProtocol": "WebDriver"
    },
    {
      "browserName": "internet explorer",
      "maxInstances": 1,
      "seleniumProtocol": "WebDriver"
    }
  ],
  "proxy": "org.openqa.grid.selenium.proxy.DefaultRemoteProxy",
  "maxSession": 5,
  "port": 6666,
  "register": true,
  "registerCycle": 5000,
  "hub": "http://localhost:4455",
  "nodeStatusCheckTimeout": 5000,
  "nodePolling": 5000,
  "role": "node",
  "unregisterIfStillDownAfter": 60000,
  "downPollingLimit": 2,
  "debug": false,
  "servlets" : [],
  "withoutServlets": [],
  "custom": {}
}

启动后在浏览器输入地址http://localhost:4455/grid/console，因为json中设置的hub是http://localhost:4455。

4 在Python中使用

4.1 使用单个节点

当配置好hub和Node后，如何用Python来启动相应的节点呢。

在def setUp(self)里使用下面方法

self.driver = webdriver.Remote(command_executor = 'http://192.168.50.115:5555/wd/hub',
                                       desired_capabilities={'platform': 'WINDOWS',
                                        'browserName':'chrome',
                                          'version': '',
                                          'javascriptEnabled': True})

有几个注意点：

  1. command_executor的端口号是Node的端口号，不是hub的端口号，因为最终测试的执行还是在Node上执行。
  2. desired_capabilities的各个参数用来配置这个测试使用哪一个操作系统和浏览器，还支持输入浏览器版本号。

4.2 使用多个节点

要使用多个节点，可以先把节点地址和浏览器创建在一个字典内，然后用for循环进行逐个使用。

# 定义node_hub与浏览器对应关系
nodes = {
    'http://192.168.50.115:5555/wd/hub': 'chrome',
    'http://192.168.50.115:6666/wd/hub': 'internet explorer',
    'http://192.168.50.115:7777/wd/hub': 'firefox'
}

# 通过不同的浏览器执行测试脚本
for host, browser in nodes.items():
    print(host, browser)
    # 调用remote方法
    driver = Remote(command_executor=host,
                    desired_capabilities={'platform': 'ANY', 'browserName': browser, 'version': '', 'javascriptEnabled': True})

driver.get("http://www.baidu.com")
driver.find_element_by_id("kw").send_keys("hello")
driver.find_element_by_id("su").click()
time.sleep(2)
driver.quit()

5. 可能存在的问题

运行时候你会发现，IE浏览器无法打开，而Chrome和Firfox却没问题。

解决办法

打开IE浏览器，Internet 选项，安全。将启用保护模式勾选取消，四个区域都要这样设置
