使用Pytest进行测试

发表于 2020-07-28 更新于 2020-11-12 分类于 Python Valine：

What is Pytest?

与unittest进行对比，pytest简直太简单，这个简单是容易阅读；容易编写；容易拓展；不仅如此，测试失败还会提醒。测试用例使用方法进行编写的，unittest是用类进行编写。

基础入门

因为参考某篇文章，对新项目创建一个虚拟环境，同一台计算机上项目就不会相互冲突，在这里使用pipenv。

全局安装pipenv

因为本机使用的是anaconda，所以打开Anaconda Prompt，输入：

pip install pipenv。

然后在项目文件中安装pytest：

pipenv install pytest --dev

这样在项目文件中就会出现两个新的文件：Pipfile Pipfile.lock

一个简单的例子

一般将测试用例都放在一个目录下面，目录名称遵守规定，以test开头。

mkdir tests
cd tests

创建一个py文件：test_match.py

def test_addition():
    assert 1 + 1 == 2

看看吧，是不是和unittest比起来简洁很多。都不用导入其他库，而且使用Python自带的assert语句就可以断言了。

运行测试

有看到两种方法运行测试：

  1. 在tests目录下执行：pyetst test_match.py;或者pytest -v test_match.py. -v可以在控制台输出更详细的信息。
  2. 回到项目的目录下执行：pipenv run python -m pytest

两者的执行结果都是一样的。

显示失败详情

如果测试失败，通过-v这个选项可以显示具体哪里失败
